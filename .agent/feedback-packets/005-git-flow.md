# Git Flow

Status: active

## User Direction

- Break every task into a focused feature branch.
- Push and merge validated feature work into `main`.
- Use `main` as the active integration line for the next version.
- Create a numbered stable branch only after enough progress is approved.
- Keep stable release branches frozen.

## Agent Behavior

Before editing, update `main` and create `feature/<goal>`. Do not create the next numbered branch early and do not commit routine development directly to `main`.
